# Vibe Coding vs AI Pair Programming

## 📌 Project Overview
This project compares two different AI-assisted development approaches:

- Vibe Coding (v0)
- AI Pair Programming (Cursor / Copilot)

The same task manager app is built using both methods and compared.

---

## 🚀 Features
- Add a task
- Mark task as complete
- Filter tasks (All / Active / Completed)
- Simple and clean UI

## ⚡ Vibe Version Observations

Vibe version was generated in around 1 minute using v0. The tool created a complete Next.js project automatically with folders like app, components, hooks, and lib. The UI was clean and functional without writing any code. However, the structure was complex and not fully understandable at first, and multiple files were generated without clear manual control.

## 🎯 When I Would Use Each Tool

**Vibe coding tool for:** quick prototypes and demos — because it generates full UI instantly.

**AI pair programming for:** production-level apps — because it provides full control and easy debugging.

---

## 📌 Conclusion
Vibe tools are fast but less controllable, while pair programming tools are slower but provide better understanding and flexibility.
